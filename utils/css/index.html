css/

  • reset.css — Box-sizing, margini zero, body base, .nascosto, .sr-only, .container
  • variabili.css — Template :root con colori, ombre, badge. Cambia i valori qui per ogni verifica.
  • layout.css — Header sticky, hero (bg-image + gradiente), griglie (prodotti / presentazione / contatti), barra filtri sticky, footer, responsive 900px e 600px
  • componenti.css — Card prodotto (+ varianti hover), card-info, card-prezzo, badge (5 colori), dropdown custom UL/LI, filtri a bottoni, input ricerca, bottoni, modal, dettaglio in-pagina, accordion, form, box-evidenziato, prezzo

Come usarli

Opzione A — Un unico style.css (consigliata per la verifica):
  Copia il contenuto di variabili.css in cima,
  poi reset.css, poi layout.css, poi componenti.css.

Opzione B — File separati con @import:
  @import url('../utils/css/variabili.css');
  @import url('../utils/css/reset.css');
  @import url('../utils/css/layout.css');
  @import url('../utils/css/componenti.css');

Hover sulle card — 4 varianti

Default (nessuna classe extra):  translateY(-0.25rem) + ombra
card--hover-ombra:               solo ombra, niente movimento
card--hover-scala:               scale(1.025) + ombra
card--hover-bordo:               outline colorato
card--hover-sfondo:              sfondo che schiarisce

Accordion — come funziona

CSS: grid-template-rows 0fr → 1fr (anima l'altezza reale, niente max-height arbitrario).
JS:  inizializzaAccordion() in componenti.js — fa solo classList.add/remove('aperto').
HTML: .accordion-corpo deve avere UN solo figlio diretto (il <p> o un <div> wrapper).

rem vs px

rem → padding, margin, gap, font-size, max-width
px  → border-width, box-shadow offset, outline (valori sotto 3px)
